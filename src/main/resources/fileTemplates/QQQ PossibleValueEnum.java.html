This template creates an enum class that implements the PossibleValueEnum interface from QQQ. This means that it has id and label fields, and a getById method (not required, by nice to have).
When you run the template, you'll be prompted for a file name (do not add the .java extension here), and your possible value's ID Type (e.g., Integer or String).
If you're using String as your idType, and your id Strings match your enum constant values, you might prefer to delete the id field and related methods, and update getPossibleValueId to return .name().
